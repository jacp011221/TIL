# 1. 변수와 타입

## 1-1. 정수 타입

허용 범위를 초과하는 값을 대입할 경우 컴파일 에러가 발생한다.

- bit: 0과 1이 저장되는 최소 단위
- n개의 bit로 구성된 정수 타입은 최상위 bit를 부호 bit로 하고, 나머지 bit가 값의 범위를 결정한다.
- 리터럴: 프로그래머가 직접 변수에 대입한 값
```java
package ch02.sec02;

public class IntegerLiteralExample {
    public static void main(String[] args) {
        int var1 = 0b1011;   // 2진수
        int var2 = 0206;     // 8진수
        int var3 = 365;      // 10진수
        int var4 = 0xB3;     // 16진수
        long varLong = 1000000000000L;

        System.out.println("var1: " + var1);         // 11
        System.out.println("var2: " + var2);         // 134
        System.out.println("var3: " + var3);         // 365
        System.out.println("var4: " + var4);         // 179
        System.out.println("varLong: " + varLong);   // 1000000000000
    }
}
```

# 2. 연산자

## 2-1. 부호/증감 연산자

- 부호 연산자: 변수의 부호를 유지하거나 변경
- 증감 연산자: 변수의 값을 1 증가 혹은 1감소
## 2-2. 산술 연산자

- 정확한 산술 연산을 원한다면 실수 타입보다는 정수 타입으로 계산하는 것이 낫다.
- 나눗셈(/)혹은 나머지(%) 연산에서 우측 피연산자가 0일 경우
## 2-3. 오버플로우/언더플로우

- 오버플로우: 타입이 허용하는 최대값을 벗어나는 것
- 언더플로우: 타입이 허용하는 최소값을 벗어나는 것
- 실행 에러는 발생하지 않으며, 최소값/최대값으로 돌아가 순환한다.
## 2-4. 비교 연산자

- 비교식을 통해 boolean 타입의 값을 반환한다.
- 피연산자의 타입이 서로 다를 경우 비교 연산을 수행하기 전 자동 변환을 통해 타입을 일치시킨다.
## 2-5. 논리 연산자

- 논리식을 통해 boolean 타입의 값을 반환한다.
## 2-6. 비트 연산자

- 정수 타입만 피연산자가 될 수 있고, 연산 결과는 int 타입(long 타입이 포함되었을 경우 long)으로 반환된다.
- 비트 논리 연산자는 1은 true, 0은 false라고 생각했을 때 논리 연산자와 동일하다.
```java
package ch03.sec08;

public class BitLogicExample {
    public static void main(String[] args) {
				int test = 136;   // 원래 값 136
        byte receiveData = (byte) test;   // 136을 byte 타입으로 강제 변환한 값 -120
        
        int unsignedInt1 = receiveData & 0xFF;   // 원래 값 136 복원
        System.out.println(unsignedInt1);

        int unsignedInt2 = Byte.toUnsignedInt(receiveData);   // 원래 값 136 복원
        System.out.println(unsignedInt2);
        
        int num1 = 1;
        int result1 = num1 << 3;
        int result2 = num1 * (int) Math.pow(2, 3);
        // Math.pow(): 2³을 연산하고, double 값을 산출
        System.out.println("result1: " + result1);
        System.out.println("result2: " + result2);

        int num2 = -5;
        int result3 = num2 >> 1;
        int result4 = num2 / 2;
        System.out.println("result3: " + result3);   // -3
        System.out.println("result4: " + result4);   // -2
    }
}
```

## 3-1. if문

- 조건식의 결과에 따라 블록 실행 여부가 결정되는 조건문
- if-else문: true이면 if문 블록 실행, false이면 else 블록 실행
- else if: if문과 else 블록 사이에 존재하는 블록으로, 상위 조건식이 false일 경우 평가되어 true일 경우 실행, false일 경우 다음 블록으로 진행
```java
package ch04.sec02;

public class IfDiceExample {

        int num = (int)(Math.random()*6) + 1;

        if(num==1) {
            System.out.println("1번이 나왔습니다.");
        } else if(num==2) {
            System.out.println("2번이 나왔습니다.");
```

## 3-2. switch문

- 괄호 안의 변수값과 일치하는 case의 실행문을 실행하는 조건문
- 업데이트
```java
package ch04.sec03;

public class SwitchValueExample {
    public static void main(String[] args) {
        String grade = "B";

        int score1 = 0;
        switch(grade) {
            case "A":
                score1 = 100;
                break;
            case "B":
                int result = 100 - 20;
                score1 = result;
                break;
            default:
                score1 = 60;
        }
        System.out.println("score1: " + score1);

        int score2 = switch(grade) {
            case "A" -> 100;
            case "B" -> {
                int result = 100 - 20;
                yield result;
            }
            default -> 60;
        };
        System.out.println("score2: " + score2);
    }
}
```

## 3-4. while문

- 조건식이 참인 동안 실행문을 반복 실행하는 반복문
```java
package ch04.sec05;

import java.util.Scanner;

public class KeyControlExample {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        boolean run = true;
        int speed = 0;

        while(run) {
            System.out.println("-------------------------");
            System.out.println("1. 증속 | 2. 감속 | 3. 중지");
            System.out.println("-------------------------");
            System.out.println("선택 : ");

            String strNum = scanner.nextLine();

            if(strNum.equals("1")) {
                speed++;
                System.out.println("현재 속도 = " + speed);
            } else if(strNum.equals("2")) {
                speed--;
                System.out.println("현재 속도 = " + speed);
            } else if(strNum.equals("3")) {
                run = false;
            }
        }

        System.out.println("프로그램 종료");
    }
}
```

## 3-5. do-while문

- 블록 내부를 실행시킨 후 조건식을 평가한다. 이 외에는 while문과 동일하다.
```java
package ch04.sec06;

import java.util.Scanner;

public class DoWhileExample {
    public static void main(String[] args) {
        System.out.println("메시지를 입력하세요.");
        System.out.println("프로그램을 종료하려면 q를 입력하세요.");

        Scanner scanner = new Scanner(System.in);
        String inputString;

        do {
            System.out.print(">");
            inputString = scanner.nextLine();
        } while( !inputString.equals("q") );

        System.out.println();
        System.out.println("프로그램 종료");
    }
}
```

### 8-1-3. 로컬 클래스

- 필드, 생성자, 메소드를 가질 수 있다.
- 로컬 클래스에 사용되는 로컬 변수는 final이어야 한다.
```java
package ch09.sec04.exam03;
```

## 8-2. 중첩 인터페이스

- 클래스의 멤버로 선언된 인터페이스
- 중첩 인터페이스는 바깥 클래스의 멤버 취급이므로 모든 접근 제한자를 사용할 수 있다.
- 기본적으로 static 특성을 가지기 때문에 해당 선언자는 생략 가능하다.
```java
package ch09.sec06.exam03;

public class Button {
    public static interface ClickListener {
        void onClick();
        }

    private ClickListener clickListener;

    public void setClickListener(ClickListener clickListener) {
        this.clickListener = clickListener;
    }

    public void click() {
        this.clickListener.onClick();
    }
}
```

```java
package ch09.sec06.exam03;

public class ButtonExample {
    public static void main(String[] args) {
        Button btnOk = new Button();

        class OkListener implements Button.ClickListener {
            @Override
            public void onClick() {
                System.out.println("Ok 버튼을 클릭했습니다.");
            }
        }

        btnOk.setClickListener(new OkListener());

        btnOk.click();

        Button btnCancel = new Button();

        class CancelListener implements Button.ClickListener {
            @Override
            public void onClick() {
                System.out.println("Cancel 버튼을 클릭했습니다.");
            }
        }

        btnCancel.setClickListener(new CancelListener());

        btnCancel.click();
    }
}
```

## 8-3. 익명 자식 객체

- 중괄호 내부의 필드/메소드는 중괄호 내부에서만 사용 가능

# 9. 라이브러리와 모듈

## 9-1. 라이브러리

- 프로그램 개발 시 활용할 수 있는 클래스/인터페이스를 모아놓은 것
- 일반적으로 JAR(Java ARchive) 압축 파일인 .jar 형태로 존재
## 9-2. 모듈

- 패키지 관리 기능까지 포함된 확장된 라이브러리
- 패키지를 선택하여 은닉할 수 있다.
- 모듈 기술자(module-info.java)에 의존 모듈을 기술하기 때문에 모듈 간의 의존 관계를 파악하기 쉽다.
- 전이 의존: A가 B를, B가 C를 의존할 때, A에서 C를 사용하고 싶다면 B에서 transitive를 통해 C를 전이 의존해야 한다.
- 집합 모듈: 자체적인 패키지가 없고, 모듈 기술자에 전의 의존 설정만 하는 모듈
- 전이 의존/집합 모듈의 사용 여부와 관계없이, 사용하는 모든 모듈은 모듈패스에 존재해야 한다.
```java
module my_module_a {
    exports pack1;     // pack1 패키지 외부에서 사용 가능
    //exports pack2;   // pack2 패키지 외부에서 사용 불가(은닉)
    
    requires my_module_b;   // my_module_b 모듈 의존
    requires transitive my_module_c;   // my_module_c 모듈 전이 의존
}
```

```java
package app;

import pack1.A;   // import해서 사용
import pack3.C;

public class Main {
    public static void main(String[] args) {
        A a = new A();
        a.method();
    }
}
```
