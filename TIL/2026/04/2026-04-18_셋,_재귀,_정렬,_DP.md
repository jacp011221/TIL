# 1. 자료구조

## 1-1. 배열(Array)

- 연속적인 메모리의 주소를 연결한 자료구조
- 데이터의 접근(읽기/쓰기)이 자주 일어나는 프로그램에 적합
## 1-2. 연결 리스트(Linked List)

- 불연속적인 메모리의 주소를 연결해서 연속적으로 보이게 만든 자료구조
- 데이터의 수정(삽입/제거)이 자주 일어나는 프로그램에 적합
```java
package linkedlist;

public class Node {
    public int data;    // Node가 가지는 데이터 값 인스턴스
    Node next;          // 다음 Node의 주소를 참조하는 인스턴스

    public Node(int data) {   // 생성자
        this(data, null);
    }
    public Node(int data, Node next) {   // 생성자
        this.data = data;
        this.next = next;
    }
}
```

```java
package linkedlist;

public class LinkedList {   // Node를 연결 리스트의 형태로 관리하는 객체
    public Node head;   // 현재 LinkedList의 시작 Node 주소를 참조하는 인스턴스
    public int count;   // 현재 LinkedList의 길이 값 인스턴스

    public LinkedList() {   // 생성자
    }

    // 모든 데이터 출력
    public void printAll() {
        Node currentNode = this.head;
        String text = "[";

        while(currentNode != null) {
            text += currentNode.data;
            currentNode = currentNode.next;
            // 다음 Node의 주소로 이동
            // 만약 이번 currentNode가 마지막 Node였다면, null로 이동하게 된다.

            if(currentNode != null) {
            // 이번 currentNode가 마지막 Node가 아니라면 ", "를 저장한다.
                text += ", ";
            }
        }

        System.out.println(text + "]");
        // 지금까지 더한 문자열을 출력한다.
    }

    // 모든 데이터 제거
    public void clear() {
        this.head = null;
        // 시작 주소를 null 값으로 돌리므로 insertAt(0, data),
        // insertLast(data) 외의 접근이 불가능해진다.
        this.count = 0;
        // 현재 LinkedList의 길이를 0으로 초기화
    }

    // 인덱스 삽입
    public void insertAt(int index, int data) throws IllegalArgumentException {
        if(index < 0 || index > this.count) {
            throw new IllegalArgumentException("범위를 넘어갔습니다.");
            // 0 ~ LinkedList의 길이를 벗어난 인덱스를 지정하면 예외가 발생한다.
        }

        Node newNode = new Node(data);   // 새 Node 생성
        if(index == 0) {   // head 위치에 Node를 연결하는 경우
            newNode.next = this.head;
            // 새 Node의 next 참조 변수를 기존의 head 주소로 지정
            this.head = newNode;
            // head 주소를 새 Node 주소로 변경
        } else {
            Node currentNode = this.head;

            for(int i=0; i<index - 1; i++) {
                currentNode = currentNode.next;
                // 다음 Node의 주소로 이동
                // 새 Node를 삽입할 인덱스의 전 Node까지 이동한다.
            }
            newNode.next = currentNode.next;
            // 새 Node의 next 참조 변수를 기존 인덱스의 주소로 지정
            //
            // 만약 Node가 3번 인덱스에 삽입되었다면,
            // 기존의 3번 인덱스 Node는 4번 인덱스로 밀려나기 때문
            currentNode.next = newNode;
            // Node 바로 앞 인덱스의 next 참조 변수를 새 Node로 변경
        }
        this.count++;
        // LinkedList의 길이 1 증가
    }

    // 마지막 인덱스 + 1 삽입
    public void insertLast(int data) {
        this.insertAt(this.count, data);
    }

    // 인덱스 제거
    public Node deleteAt(int index) throws IllegalArgumentException {
        if(index < 0 || index > this.count - 1) {
            throw new IllegalArgumentException("제거할 수 없습니다.");
            // 0 ~ (LinkedList의 길이 - 1)을 벗어난 인덱스를 지정하면 예외가 발생한다.
        }

        if(index == 0) {   // head 위치의 Node를 제거하는 경우
            Node deleteNode = this.head;
            // return 값으로 반환하기 위해 제거할 Node 저장
            this.head = this.head.next;
            // head 주소를 기존 head 주소의 next 주소로 변경
            this.count--;   // LinkedList의 길이 1 감소
            return deleteNode;
        } else {
            Node currentNode = this.head;
            
            for(int i=0; i<index - 1; i++) {
                currentNode = currentNode.next;
                // 다음 Node의 주소로 이동
                // 제거할 Node의 전 Node까지 이동한다.
            }
            Node deleteNode = currentNode.next;
            // return 값으로 반환하기 위해 제거할 Node 저장
            currentNode.next = deleteNode.next;
            // 전 노드의 next 주소를 제거할 Node의 next 주소로 변경
            this.count--;   // LinkedList의 길이 1 감소
            return deleteNode;
        }
    }

    // 마지막 인덱스 제거
    public Node deleteLast() {
        return this.deleteAt(this.count - 1);
    }

    // 인덱스 데이터 반환
    public Node getNodeAt(int index) throws IllegalArgumentException {
        if(index < 0 || index > this.count - 1) {
            throw new IllegalArgumentException("범위를 넘어갔습니다.");
            // 0 ~ (LinkedList의 길이 - 1)을 벗어난 인덱스를 지정하면 예외가 발생한다.
        }
        
        Node currentNode = this.head;
        
        for(int i=0; i<index; i++) {
            currentNode = currentNode.next;
            // 다음 Node의 주소로 이동
            // 반환할 Node까지 이동한다.
        }

        return currentNode;
    }
}
```

## 1-3. 스택(Stack)

- Last In First Out(LIFO, 후입선출) 특성을 갖는 리스트형 자료구조
- 문법검사기의 괄호 짝 검사, 작업 프로그램의 돌아가기(Ctrl+Z) 등에 주로 사용된다.
```java
package stack;

import linkedlist.*;

public class Stack {
    LinkedList list;

    public Stack() {
        this.list = new LinkedList();
    }

    public void push(int data) {
        this.list.insertAt(0, data);
    }

    public Node pop() {
        try{
            return this.list.deleteAt(0);
        } catch (IllegalArgumentException e) {
            return null;
        }
    }

    public Node peek() {
        return this.list.head;
    }

    public boolean isEmpty() {
        return (this.list.count == 0);
    }
}
```

## 1-4. 큐(Queue)

- First In First Out(FIFO, 선입선출) 특성을 갖는 자료구조
- 양방향 연결 리스트로 구현할 수 있다.
## 1-5. 덱(Deque)

- 양쪽 끝에서 수정(삽입/제거)이 가능한 자료구조
- 양방향 연결 리스트로 구현할 수 있다.
## 1-6. 해시 테이블(Hash Table)

- 해시 함수를 사용해 키(Key)를 인덱스화하여 데이터를 저장하는 자료구조
## 1-7. 셋(Set)

- 데이터의 중복을 허용하지 않는 자료구조
# 2. 알고리즘

## 2-1. 재귀(Recursion)

- 자기 자신을 참조하는 함수
## 2-2. 정렬(Sort)

### 2-2-1. 버블 정렬(Bubble Sort)

- 정렬되지 않은 영역(왼쪽)에서 왼쪽 원소가 더 크면 오른쪽 원소와 자리를 바꾸는 과정(n)을 (n-1)번 반복하는 정렬 (O(n²))
### 2-2-2. 선택 정렬(Selection Sort)

- 왼쪽부터 배열을 전부 순회해 가장 작은 원소를 선택한 후 정렬되지 않은 영역(오른쪽)의 첫 번째 원소와 교환하는 과정(n)을 (n-1)번 반복하는 정렬 (O(n²))
### 2-2-3. 삽입 정렬(Insertion Sort)

- 정렬되지 않은 원소를 정렬된 영역(왼쪽)의 적절한 위치에 삽입하는 정렬 (O(n²))
### 2-2-4. 병합 정렬(Merge Sort)

- 반씩 분할 정복(logn)하는 정렬(O(nlogn))
### 2-2-5. 퀵 정렬(Quick Sort)

- 피벗을 설정하고 피벗을 기준으로 왼쪽 영역과 오른쪽 영역으로 나눠 분할 정복하는 정렬(Θ(nlogn), O(n²))
## 2-3. 동적 프로그래밍(Dynamic Programming, DP)

- 문제를 작은 단위로 쪼개 해결하되, 중복되는 부분 문제를 저장하여 재사용하는 방식
### 2-3-1. 메모이제이션(Memoization)

- 하향식 계산에서, 중복 계산을 줄이기 위해 결과를 저장하는 방식
### 2-3-2. 타뷸레이션(Tabulation)

- 상향식 계산에서, 이전 계산을 기반으로 다음 계산을 진행하는 방식
